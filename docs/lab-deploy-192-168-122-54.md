# Lab Deploy — Auth + App VPS

Triển khai 2 VPS lab:

| VPS | IP | Vai trò | Stack |
|---|---|---|---|
| **VPS-A (Auth)** | `192.168.122.54` | Zitadel central (OIDC issuer) | `infra/auth-vps/` |
| **VPS-B (App)** | `192.168.122.55` | IAP reference (oauth2-proxy + sample apps) | `infra/app-vps/` + `sample-apps/` |

**Goal**: Up Zitadel self-signed cert + 2 demo app phía sau oauth2-proxy → validate workflow OIDC login end-to-end. Sau khi soak ổn → cơ sở promote production.

**Pattern**: Identity-Aware Proxy. App downstream KHÔNG code OIDC, đọc identity qua `X-Auth-Request-*` headers từ oauth2-proxy.

Generate secrets (oneliner — tránh `!` bash history expansion):

```bash
set +H
MK=$(openssl rand -base64 32 | tr -d '\n=+/' | cut -c1-32)
PGP=$(openssl rand -base64 24)
ZDB=$(openssl rand -base64 24)
ZAP="$(openssl rand -base64 16 | tr -d '/+=')@Aa1"   # @ thay ! để tránh history expansion

sed -i "s|ZITADEL_MASTERKEY=.*|ZITADEL_MASTERKEY=${MK}|" .env
sed -i "s|POSTGRES_ADMIN_PASSWORD=.*|POSTGRES_ADMIN_PASSWORD=${PGP}|" .env
sed -i "s|ZITADEL_DB_PASSWORD=.*|ZITADEL_DB_PASSWORD=${ZDB}|" .env
sed -i "s|ZITADEL_ADMIN_PASSWORD=.*|ZITADEL_ADMIN_PASSWORD=${ZAP}|" .env

# LƯU VÀO PASSWORD MANAGER NGAY
echo "MASTERKEY:    ${MK}"
echo "POSTGRES:     ${PGP}"
echo "ZITADEL_DB:   ${ZDB}"
echo "ADMIN:        ${ZAP}"

# Verify ADMIN đủ 4 nhóm ký tự
for c in '[a-z]:lower' '[A-Z]:upper' '[0-9]:digit' '[^a-zA-Z0-9]:symbol'; do
  echo "${ZAP}" | grep -q "${c%:*}" && echo "${c#*:} OK"
done
```

**Backup masterkey ≥2 nơi** (Bitwarden + sealed envelope). Mất masterkey = backup vô dụng.

Fill **đúng format** — value 1 dòng sạch, KHÔNG inline comment sau `=`:

```dotenv

ZITADEL_IP=192.168.122.54

NEXTJS_HOSTNAME=nextjs-demo.lab.local
NEXTJS_CLIENT_ID=<paste Client ID từ Zitadel app nextjs-demo>

STATIC_CLIENT_SECRET=<paste Client Secret>
STATIC_COOKIE_SECRET=
```

> **Critical**: KHÔNG để comment trên cùng dòng với value. `.env` parser tính TOÀN BỘ phần sau `=` là value, kể cả `# comment`.

Generate 2 cookie secret:

```bash
set +H
N1=$(openssl rand -base64 32 | tr -d '\n=+/' | cut -c1-32)
N2=$(openssl rand -base64 32 | tr -d '\n=+/' | cut -c1-32)
sed -i "s|^NEXTJS_COOKIE_SECRET=.*|NEXTJS_COOKIE_SECRET=${N1}|" .env
sed -i "s|^STATIC_COOKIE_SECRET=.*|STATIC_COOKIE_SECRET=${N2}|" .env

# Verify TẤT CẢ field có giá trị
grep -E '^(NEXTJS|STATIC)_(CLIENT_ID|CLIENT_SECRET|COOKIE_SECRET)=' .env \
  | awk -F= '{print $1"="(length($2)>0?"<set "length($2)" chars>":"<EMPTY>")}'
# Expect:
#   CLIENT_ID: ~18 chars (số)
#   CLIENT_SECRET: 40-64 chars
#   COOKIE_SECRET: 32 chars
```

Nếu có `<EMPTY>` → fill lại bằng tay.

# Khác biệt Lab vs Production

| Aspect | Lab | Production |
|---|---|---|
| TLS | Self-signed | Let's Encrypt qua ACME DNS-01 |
| Domain | `*.lab.local` qua /etc/hosts | DNS thật, parent domain riêng từng app |
| SMTP | mailhog | SMTP relay (Mailgun/SES/công ty) |
| Backup offsite | Skip | rclone → S3 nội bộ, cron daily |
| Restore drill | Manual | Cron weekly |
| Monitoring | Skip | Uptime Kuma + Telegram alert |
| Break-glass admin | Optional | BẮT BUỘC (founder/sếp) |
| MFA | TOTP (cert self-signed block Passkey) | Passkey + TOTP backup |
| oauth2-proxy `--ssl-insecure-skip-verify` | true (Zitadel self-signed) | false |

# Troubleshooting (tổng hợp issue thực tế đã gặp)

## Pre-install (cả 2 VPS)
| Symptom | Root cause | Fix |
|---|---|---|
| `sudo: unable to resolve host <name>` warning loop | hostname không có trong /etc/hosts | `echo "127.0.1.1 $(hostname)" \| sudo tee -a /etc/hosts` |
| `git clone` báo `Permission denied` trong `/opt` | `/opt` cần root | `sudo chown $USER:$USER /opt/authway` trước khi clone |
| `ufw allow port 80,443` syntax error | ufw không hỗ trợ list ports trên 1 dòng | Tách: 2 lệnh ufw allow riêng |

## Port conflicts (cả 2 VPS)
| Symptom | Root cause | Fix |
|---|---|---|
| Traefik fail bind 0.0.0.0:80/443 | nginx/apache/litespeed/cyberpanel chiếm sẵn | stop + disable + **mask** + pkill |
| Service start lại sau stop | systemd chưa mask | `sudo systemctl mask <name>` |

## VPS-B từ snapshot
| Symptom | Root cause | Fix |
|---|---|---|
| VPS-B có sẵn containers authway-auth-* | Snapshot/clone từ VPS-A | `docker ps -a --format '{{.Names}}' \| grep authway-auth- \| xargs docker rm -fv` |
| `ping auth.lab.local` → 127.0.0.1 trên VPS-B | /etc/hosts kế thừa từ VPS-A | `sudo sed -i '/^127\.0\.0\.1 auth\.lab\.local$/d' /etc/hosts` |

## Database init/setup (VPS-A only)
| Symptom | Root cause | Fix |
|---|---|---|
| Postgres unhealthy + `POSTGRES_USER not set` warning | Healthcheck dùng env var khác container | Healthcheck dùng `$${POSTGRES_USER}` (in-container) |
| `zitadel-init` fail `user "application_name=zitadel_queries"` | Env var Zitadel không parse đúng | Render YAML qua envsubst (`scripts/render-config.sh`) |
| `zitadel-setup` exit 1 `HasSymbol`/`HasLower` | Admin password thiếu nhóm ký tự | `"$(openssl rand -base64 16 \| tr -d '/+=')@Aa1"` + `set +H` |
| `zitadel-setup` migration fail giữa chừng | DB volume dirty từ lần fail | `docker compose down -v` (chỉ khi chưa có data thật) |

## Traefik routing (cả 2 VPS)
| Symptom | Root cause | Fix |
|---|---|---|
| `docker compose logs traefik` trống | Log vào file | Bỏ `filePath` trong `traefik.yml` → stdout |
| `client version 1.24 is too old. Minimum 1.40` | Daemon API min quá cao cho Traefik image cũ | A: `image: traefik:v3.4`. B: `/etc/docker/daemon.json` min `1.24` + restart docker |
| `Filtering unhealthy or starting container` | Healthcheck chưa pass → Traefik skip | Đợi ~60s. Healthcheck zitadel cần `TLS.Enabled: false` trong config |
| `/api/http/routers` chỉ thấy `*@internal` | Provider docker chưa connect | Check `docker compose logs traefik` |

## Zitadel healthcheck (VPS-A only)
| Symptom | Root cause | Fix |
|---|---|---|
| `zitadel ready` báo `http: server gave HTTP response to HTTPS client` | Zitadel internal HTTP, ready default HTTPS | `TLS: { Enabled: false }` trong `zitadel-config.yaml` |
| `zitadel ready` báo `unknown flag: --tlsMode` | Flag không support cho `ready` subcommand | Bỏ flag, dùng `TLS.Enabled` trong config |

## oauth2-proxy (VPS-B only)
| Symptom | Root cause | Fix |
|---|---|---|
| `missing setting: cookie-secret` | `.env` cookie secret empty | Generate 32 chars: `openssl rand -base64 32 \| tr -d '/+=' \| cut -c1-32` |
| `.env` field giá trị dài bất thường (>50 chars cho client_id) | Inline comment `#` sau `=` bị tính vào value | KHÔNG để comment cùng dòng với value |
| `lookup auth.lab.local on 127.0.0.11:53: no such host` | Container DNS không thấy /etc/hosts của host | Thêm `extra_hosts` mapping `ZITADEL_HOSTNAME:ZITADEL_IP` trong compose |
| Trang app hiện ra với identity = null/missing | oauth2-proxy bị bypass — request không qua proxy | Verify Traefik router service trỏ vào oauth2-proxy (`nextjs@docker → nextjs` service port 4180) |
| `redirect_uri mismatch` lúc login | Zitadel app config khác với redirect_url của oauth2-proxy | Update Redirect URI trong Zitadel app: `https://<HOSTNAME>/oauth2/callback` |
| `Authentication Method: None` không có Client Secret | OIDC app config PKCE-only | Đổi sang **Basic** trong Zitadel → secret hiện 1 lần |

## Browser / cert
| Symptom | Fix |
|---|---|
| `ERR_CERT_AUTHORITY_INVALID` | Import `.crt` vào trust store hoặc accept exception |
| `WebAuthn not supported on sites with TLS certificate errors` | Self-signed cert block Passkey → dùng TOTP cho lab. Production có cert thật thì OK |
| Login redirect loop | `EXTERNAL_DOMAIN` trong `.env` phải match exact với URL trên browser |
| Browser vào thẳng trang sau khi clear cookie (vẫn không redirect login) | Browser cache / Service Worker / HSTS preload | DevTools → Application → Storage → Clear site data. Hoặc `chrome://net-internals/#hsts` → Delete domain. Hoặc test **incognito** |
