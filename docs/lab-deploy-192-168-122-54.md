# Lab Deploy — VPS 192.168.122.54

Bước-by-bước deploy Zitadel auth central lên VPS lab `192.168.122.54` (private network, không có Internet ACME).

**Goal**: Up Zitadel với self-signed cert, mailhog SMTP, không offsite backup (chỉ local), test login + MFA enrollment. Sau đó mới có cơ sở proceed phase 02.

---

## 0. Pre-flight (máy của bạn)

- SSH access vào `192.168.122.54` (root hoặc sudo user)
- Có thể resolve được `192.168.122.54` từ máy bạn (cùng VPN/network hoặc routing)
- Browser dùng để test có thể tới `192.168.122.54`

## 1. Provision VPS (SSH vào 192.168.122.54)

```bash
ssh vietnt@192.168.122.54   # hoặc user sudo của bạn

# Fix sudo hostname warning (nếu hostname chưa có trong /etc/hosts)
echo "127.0.1.1 $(hostname)" | sudo tee -a /etc/hosts

# OS update
sudo apt update && sudo apt upgrade -y

# Docker + compose plugin
curl -fsSL https://get.docker.com | sh
sudo apt install -y docker-compose-plugin git rclone openssl curl

# Add user vào docker group để chạy docker không cần sudo
sudo usermod -aG docker $USER
newgrp docker
docker ps   # verify hoạt động, không cần sudo

# NTP — critical cho TOTP (red-team note: NTP drift = mass MFA fail)
sudo timedatectl set-ntp true
timedatectl status   # Expect: "NTP service: active", "System clock synchronized: yes"

# Firewall — chỉ allow 80/443 từ private subnet
sudo ufw allow from 192.168.122.0/24 to any port 80 proto tcp
sudo ufw allow from 192.168.122.0/24 to any port 443 proto tcp
sudo ufw allow from 192.168.122.0/24 to any port 22 proto tcp
sudo ufw default deny incoming
sudo ufw --force enable

# Verify từ outside subnet:
# curl -k https://192.168.122.54  → connection refused/timeout (expected)
```

## 2. Clone repo + chuẩn bị config

```bash
# /opt cần root → chown về user để git clone không cần sudo
sudo mkdir -p /opt/authway
sudo chown $USER:$USER /opt/authway
git clone https://github.com/nguyenviet2509/authway.git /opt/authway
cd /opt/authway/infra/auth-vps

# Tạo .env
cp .env.example .env
nano .env
```

Fill `.env` cho lab:

```dotenv
ZITADEL_EXTERNAL_DOMAIN=auth.lab.local

# Generate 3 secrets:
#   openssl rand -base64 32 | tr -d '\n' | head -c 32   → MASTERKEY
#   openssl rand -base64 24                              → 2 passwords
ZITADEL_MASTERKEY=<paste>
POSTGRES_ADMIN_USER=postgres
POSTGRES_ADMIN_PASSWORD=<paste>
ZITADEL_DB_USER=zitadel
ZITADEL_DB_PASSWORD=<paste>
ZITADEL_ADMIN_USERNAME=zitadel-admin
ZITADEL_ADMIN_PASSWORD=<paste>

# Lab: bỏ trống S3, telegram — disable monitoring stack
S3_ENDPOINT=
S3_BUCKET=
S3_ACCESS_KEY=
S3_SECRET_KEY=
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=

# Lab SMTP via mailhog
SMTP_HOST=mailhog
SMTP_PORT=1025
SMTP_USER=
SMTP_PASSWORD=
SMTP_FROM=noreply@lab.local
```

**Backup masterkey**: copy `ZITADEL_MASTERKEY` value ra ≥2 nơi (password manager + file riêng) — kể cả lab vẫn nên habit này.

## 3. Self-signed TLS cert

```bash
cd /opt/authway/infra/auth-vps
chmod +x scripts/*.sh
bash scripts/generate-lab-cert.sh
# → tạo tls/auth.crt + tls/auth.key cho auth.lab.local + IP 192.168.122.54
```

## 4. DNS / hosts entry

**Trên VPS**:
```bash
echo "127.0.0.1 auth.lab.local" | sudo tee -a /etc/hosts
```

**Trên máy client (máy bạn dùng để mở browser test)**:
- Linux/Mac: `sudo nano /etc/hosts` → thêm `192.168.122.54 auth.lab.local`
- Windows: edit `C:\Windows\System32\drivers\etc\hosts` (admin) → thêm `192.168.122.54 auth.lab.local`

Verify: `ping auth.lab.local` → resolve 192.168.122.54.

## 5. Trust self-signed cert trên client (optional, tránh red-browser warning)

Copy `tls/auth.crt` từ VPS về máy client:
```bash
scp root@192.168.122.54:/opt/authway/infra/auth-vps/tls/auth.crt ./authway-lab.crt
```

- **Linux**: `sudo cp authway-lab.crt /usr/local/share/ca-certificates/ && sudo update-ca-certificates`
- **Mac**: Keychain Access → System → drag `.crt` → set "Always Trust"
- **Windows**: `certutil -addstore -user Root authway-lab.crt`

Sau đó **restart browser**.

(Nếu skip step này: browser sẽ warn "Not Secure" — accept exception cho lab cũng được.)

## 6. Bring up stack

```bash
cd /opt/authway/infra/auth-vps
docker compose up -d

# Theo dõi init/setup (must completed_successfully)
docker compose logs -f zitadel-setup
# Khi thấy "setup completed" → Ctrl+C
docker compose logs -f zitadel
# Khi thấy "server is listening" → Ctrl+C
```

Verify services:
```bash
docker compose ps
# Expect: postgres healthy, zitadel running, traefik running, mailhog running
#         zitadel-init + zitadel-setup status=exited (0)
```

## 7. First login + bind Passkey

1. Mở browser: `https://auth.lab.local/ui/console`
2. Login với `ZITADEL_ADMIN_USERNAME` + `ZITADEL_ADMIN_PASSWORD`
3. Buộc đổi password → đặt password mạnh, lưu password manager
4. **Bind Passkey** ngay (Account → Security → Authentication factors → Add) — dùng platform authenticator (Touch ID / Windows Hello / YubiKey)
5. Logout → login lại → verify Passkey hoạt động

## 8. Tạo break-glass admin (founder/sếp)

1. Console → Users → + New
2. Username: `breakglass-<name>`, Email: founder@lab.local, set initial password
3. Sau khi tạo: Authorizations tab → Add → Role `IAM_OWNER`
4. Founder login lần đầu → đổi password + bind Passkey của founder
5. **Seal credential**: ghi username + initial password + Passkey device info vào envelope, dán seal, cất safe
6. Document break-glass procedure: khi nào dùng, ai approve, audit log review sau khi dùng

## 9. SMTP config (Zitadel → mailhog)

1. Console → Instance → SMTP Settings
2. Host: `mailhog`, Port: `1025`, From: `noreply@lab.local`, From name: `Authway Lab`
3. **No auth**, no TLS (mailhog dev mode)
4. Test send → check mailhog UI:
   - SSH tunnel: `ssh -L 8025:127.0.0.1:8025 root@192.168.122.54`
   - Mở `http://localhost:8025` → thấy email test

## 10. Tạo Project + 2 OIDC App skeleton (cho phase 02)

1. Console → Projects → + New → Name: `internal-apps`
2. Trong project → Applications → + New OIDC:
   - **nextjs-demo**: Web, Code+PKCE, Redirect URI tạm `https://placeholder/oauth2/callback` (fill thật ở phase 02)
   - **static-demo**: Web, Code+PKCE, Redirect URI tạm `https://placeholder/oauth2/callback`
3. Note `client_id` + `client_secret` mỗi app — lưu password manager (sẽ cần khi phase 02)

## 11. Verify policies (red-team checklist)

Console → Default Settings → Login Behavior:
- [x] Force MFA: **ON**
- [x] Allow Username Password: ON
- [x] Allow Register: **OFF**
- [x] Allow External IDP: **OFF**
- [x] Hide Password Reset: **ON** (red-team #9 — disable self-service)
- [x] Passwordless: **Allowed**
- [x] Second Factors: TOTP + U2F (Passkey)

Console → Default Settings → Lockout:
- [x] Max Password Attempts: 5

Test:
1. Tạo user dummy `test1@lab.local`
2. Login với user mới → buộc setup MFA flow
3. Login sai password 6 lần → bị lockout
4. Reset MFA cho test1 → chỉ admin làm được, không có self-service link trong email

## 12. Local backup (lab — không có S3)

```bash
# Manual backup vào /var/backups/authway
mkdir -p /var/backups/authway
docker compose exec -T postgres pg_dump -U postgres -d zitadel \
  --no-owner --clean --if-exists \
  | gzip > /var/backups/authway/zitadel-lab-$(date +%F).sql.gz
ls -lh /var/backups/authway/
```

(Skip rclone S3 step cho lab. Production sẽ enable cron + rclone offsite.)

## Khác biệt Lab vs Production

| Aspect | Lab (192.168.122.54) | Production |
|---|---|---|
| TLS | Self-signed (`generate-lab-cert.sh`) | Let's Encrypt qua ACME DNS-01 |
| Domain | `auth.lab.local` qua /etc/hosts | `auth.<company>.com` qua DNS thật |
| SMTP | mailhog (dev) | SMTP relay (Mailgun/SES/công ty) |
| Backup offsite | Skip (local only) | rclone → S3 nội bộ, cron daily |
| Restore drill | Manual | Cron weekly |
| Monitoring | Skip Uptime Kuma/Telegram | Full stack + Telegram alert |
| Break-glass | Optional cho lab | BẮT BUỘC (founder/sếp) |
| Firewall | Private subnet 192.168.122.0/24 | VPN-only subnet |

Khi promote từ lab → production: thay cert + domain + SMTP + bật backup/monitoring + tạo break-glass thực sự. Đừng promote masterkey/admin password từ lab — generate mới.

## Troubleshooting nhanh

| Symptom | Fix |
|---|---|
| `zitadel-setup` lặp lại restart | Check `docker compose logs zitadel-setup` — thường là DB connection hoặc masterkey ≠ 32 chars |
| Browser báo `ERR_CERT_AUTHORITY_INVALID` | Import `tls/auth.crt` vào trust store (step 5) hoặc accept exception |
| Login redirect loop | Kiểm tra `ZITADEL_EXTERNAL_DOMAIN` trong `.env` match với hosts entry và URL bạn truy cập |
| `ZITADEL_EXTERNALSECURE` warning | Đảm bảo luôn truy cập qua `https://` không phải `http://` |
| Postgres OOM | Lab tăng RAM, hoặc tune `shared_buffers` trong Postgres |
| MFA fail "code invalid" | NTP — chạy lại `timedatectl set-ntp true` và đợi sync |
