# Auth VPS — Zitadel Central Stack

Self-hosted Zitadel + Postgres + Traefik + monitoring. Deploy onto **1 dedicated VPS** (16 GB RAM). Phase 01 của plan [zitadel-iap-rollout](../../plans/260626-1154-zitadel-iap-rollout/plan.md).

## Files

| Path | Purpose |
|---|---|
| `docker-compose.yml` | Main stack: traefik + zitadel-init/setup/runtime + postgres + mailhog |
| `.env.example` | Copy → `.env`, fill secrets |
| `zitadel-config.yaml` | Zitadel runtime config + Login/Lockout policy |
| `zitadel-steps.yaml` | First-instance bootstrap |
| `traefik.yml` | Traefik static config |
| `dynamic/middlewares.yml` | Rate-limit, security headers, TLS options |
| `scripts/generate-lab-cert.sh` | Self-signed cert cho lab |
| `scripts/backup-postgres.sh` | Daily backup → S3 (cron) |
| `scripts/restore-drill.sh` | Weekly restore test (cron) |
| `scripts/check-health.sh` | 10-min probes → Telegram alert |
| `monitoring/docker-compose.yml` | Uptime Kuma + health-checker |
| `tls/` | Cert files (gitignore — generated by script) |

## Critical security notes

1. **Masterkey** lưu ≥2 nơi độc lập. Mất = backup vô dụng.
2. **≥2 admin** Passkey-enrolled (primary + break-glass founder).
3. **Self-service password/MFA reset** disabled — admin-mediated only.
4. **Postgres** bind internal Docker network, KHÔNG expose host port.
5. **Traefik dashboard** chỉ bind 127.0.0.1 — SSH tunnel để access.
6. **NTP** synced bắt buộc (TOTP).
7. **Backup cron + restore drill** từ ngày 1, không phải khi production.
